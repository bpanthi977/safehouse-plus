# UI design 01: Run screen

## First screen: Run tab

```text
safehouse+

 Run  |  Network  |  Paths  |  Environment  |  Features
────────────────────────────────────────────

Command
  pi

Actions
> [run]  Run / Restart command
  [edit] Edit command
  [export] Write generated policy to file
  [save] Save and Quit
  [discard] Discard and Quit

Generated sandbox command
  safehouse --workdir=/Users/bpanthi977/Dev/safehouse-plus --append-profile=<generated-profile> -- pi

────────────────────────────────────────────
Enter/Space activate · ←/→ switch tabs · Ctrl-p/Ctrl-n move · q/Ctrl-C save · D discard
```

## Run tab when a resume command is detected

When a resume command is detected, it should become the default selected action.
The detected resume command is shown directly under the resume action, not in a separate Session section.

```text
safehouse+

 Run  |  Network  |  Paths  |  Environment  |  Features
────────────────────────────────────────────

Command
  pi

Actions
> [resume] Resume detected session
           pi --session abc123...
  [run]    Run / Restart command
  [edit]   Edit command
  [export] Write generated policy to file
  [save]   Save and Quit
  [discard] Discard and Quit

Generated sandbox command
  safehouse --workdir=/Users/bpanthi977/Dev/safehouse-plus --append-profile=<generated-profile> -- pi

────────────────────────────────────────────
Enter/Space activate · ←/→ switch tabs · Ctrl-p/Ctrl-n move · q/Ctrl-C save · D discard
```

## Design notes

- First screen is focused only on command/session actions.
- Editable/action rows are clearly tagged: `[run]`, `[resume]`, `[edit]`, `[export]`, `[save]`, `[discard]`.
- Non-editable rows, if added later, should be tagged `[info]`.
- If a resume command is available, `[resume]` is the default selected action.
- The resume command is shown immediately below the `[resume]` row.
- Tabs have no numeric prefixes.
- Left/right arrows switch tabs.
- No extra help line at the top.
- `[export] Write generated policy to file` prompts for an output path and writes the generated sandbox policy there.
- Save behavior is explicit: `Save and Quit`, not just `q`.
- The generated command always includes `--append-profile=<generated-profile>`. Even with no user changes, `safehouse+` emits a small profile that denies the sandboxed command access to `<workdir>/.safehouse-plus.json`, so the placeholder is always shown. The live runner substitutes a real temp-file path for `<generated-profile>`.
